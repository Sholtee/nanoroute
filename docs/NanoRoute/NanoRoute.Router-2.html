Remarks

Routes are registered as path patterns. Literal segments are matched case-insensitively, while parameter segments use parsers registered through AddParameterParser(string, ParameterParserDelegate). A pattern ending with / acts as a prefix route, so it also matches longer paths that start with the same segments. A pattern without a trailing slash is treated as an exact match.

When multiple handlers match the same request, they are invoked as a pipeline. Each handler receives a RequestContext<TRequest> and a next delegate. Calling next() transfers control to the next compatible handler; returning directly short-circuits the pipeline.

When multiple handlers match the same request, the router evaluates the shortest compatible prefix first, then continues toward more specific matches. At the same path depth, literal segment matches are preferred over parameter matches. This allows broader prefix handlers to populate Parameters before more specific handlers continue the pipeline. 

sealed class MyRouter : Router<HttpRequest, IResult>
{
    public IResult Route(HttpRequest request, IServiceProvider services) => Handle(request, services);

    protected override Uri GetUri(HttpRequest request) => request.Url;
    protected override string GetVerb(HttpRequest request) => request.Method;
    protected override IResult CreateJsonResponse(HttpStatusCode statusCode, string content) =>
        Results.Text(content, "application/json", statusCode: (int) statusCode);
}

MyRouter router = new MyRouter();

router
    .AddDefaultParsers()
    .AddHandler("GET", "/api/users/{user_id:int}/", (context, next) =>
    {
        object user = LoadUser((int) context.Parameters["user_id"]!);
        context.Parameters["User"] = user;
        return next();
    })
    .AddHandler("GET", "/api/users/{user_id:int}/details", (context, next) =>
    {
        return Results.Ok(context.Parameters["User"]);
    });

IResult response = router.Route(request, services);

In this example, a request for /api/users/42/details first matches the prefix handler, which parses and stores user_id, then continues to the more specific handler that returns the response.

Returns
Type Description
Router<TRequest, TResponse>

The current router instance.
Remarks

The default handler is registered as a prefix route for all supported HTTP methods. It calls the next matching handler and intercepts the terminal not found case as well as unhandled exceptions. Responses are created through CreateJsonResponse(HttpStatusCode, string).

Examples
router
    .AddDefaultHandler()
    .AddHandler("GET", "/health", (context, next) => Results.Ok());

In this example, requests without a matching route receive the built-in JSON 404 Not Found response instead of an unhandled exception.
