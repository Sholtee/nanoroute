Syntax
public class RouteBuilder : RoutingContext
Remarks

Route patterns support literal segments and parser-backed parameter segments such as /users/{id:int}. Consecutive / separators are treated as a single separator. A trailing / marks the pattern as a prefix match, while patterns without a trailing slash must match the full path exactly.
